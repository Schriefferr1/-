# 实验报告：对称密码算法与SM3杂凑算法软件优化实现

## 作业 3：对称密码算法的软件实现（以 AES 为例）

### 1. 实验内容
基于 C 语言以及 x86 SIMD 内联汇编指令，实现并优化 AES-128 对称密码算法，涵盖 T-table 软件优化、专用硬件指令集优化，以及针对 CTR、GCM、XTS 三种主流工作模式的并发与算术加速。

### 2. 核心优化策略与实现

*   **T-table（查表法）软件优化**：
    *   代码预计算了4个4KB大小的T表（T0, T1, T2, T3）[cite: 4]。
    *   利用查表和32位整数异或操作（如 `T0[s0 & 0xff] ^ ...`）直接完成了一轮核心操作，大幅代替了原本复杂的位移和有限域乘法[cite: 4]。
*   **AES-NI 指令集与 CTR 模式并行处理**：
    *   利用 x86 的 AES-NI 指令（如 `_mm_aesenc_si128`），实现了加密轮的硬件级加速[cite: 4]。
    *   在 CTR 工作模式中，代码通过 `aes_ctr_encrypt_4blocks` 函数同时处理 4 个数据块（64 Bytes），打破了数据依赖，实现了多块并行加密[cite: 4]。
*   **GCM 模式中的 GHASH 乘法加速**：
    *   采用 PCLMULQDQ 指令集（`_mm_clmulepi64_si128`）实现了无进位乘法[cite: 4]。
    *   结合 Karatsuba 算法降低乘法次数，并在最后使用了 `0x87` 多项式进行 GF(2^128) 的约化[cite: 4]。
*   **XTS 模式下的 Tweak 递推优化**：
    *   利用向量指令实现了 GF(2^128) 域上乘以 alpha (x) 的快速 tweak 递推操作（`xts_mult_alpha`），并高效处理了跨 64 位的进位与条件异或多项式逻辑[cite: 4]。

## 作业 4：SM3 杂凑算法的软件实现与优化

### 1. 实验内容
针对国密 SM3 杂凑算法的特点，实现基于 SIMD 寄存器和通用寄存器（GPR）混合调度的算法优化。本报告重点分析 ARM64 架构下基于 NEON 指令集的混合优化方案。

### 2. 混合架构优化实现详情

*   **大端数据加载与字节序转换**：
    *   在针对 ARM64 架构的代码中，实现了基于 NEON 向量指令与通用寄存器（GPR）混合的优化方案[cite: 5]。
    *   利用 NEON SIMD 寄存器（如 `vld1q_u32`）一次性并行加载 4 个 32 位字的数据[cite: 5]。
    *   随后通过 `vrev32q_u8` 指令完成了字节序的反转，极大地提高了初始数据处理速度[cite: 5]。
*   **消息扩展与压缩函数计算的分工**：
    *   混合分工的第一阶段主要针对消息扩展，生成 $W[16..67]$ 和 $W'[0..63]$[cite: 5]。
    *   第二阶段，将数据交回 ARM 硬件的通用寄存器（GPR，如 A, B, C 等）进行 64 步的串行压缩函数迭代计算[cite: 5]。
    *   压缩函数按照 $j=0..15$ 和 $j=16..63$ 分段展开，使用了 `FF0`, `GG0`, `FF1`, `GG1` 等宏定义计算布尔函数，并结合 `ROTL` 循环移位宏完成状态更新[cite: 5]。
